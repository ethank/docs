## Overview

Generates synthesizer presets from natural language descriptions, making complex synths accessible to everyone.

**User Flow:**
- Producer: "Create me a warm analog bass, thick, slightly distorted"
- AI generates preset parameters for their synth
- User can tweak or regenerate
- Saves as custom preset

## Technical Approach

### How It Works

```
Natural Language Description
    ↓
LLM maps to synthesis parameters
    ↓
JSON parameter output
    ↓
Apply to synth (via SDK/API)
    ↓
User preview and save
```

### Parameter Mapping

**Musical Descriptors → Synth Parameters:**

| Description | Synth Parameters |
|-------------|------------------|
| "Warm analog bass" | Filter cutoff low, oscillator waveform saw, envelope slow |
| "Bright aggressive lead" | Filter cutoff high, resonance up, envelope fast attack |
| "Thick pad" | Multiple oscillators, detuned, long release |
| "Plucky synth" | Fast attack, short decay, high velocity sensitivity |

### Starting Simple

**MVP (Week 1-2):**
- Start with ONE synth (Moog Minimoog)
- Text-only generation (no audio preview in MVP)
- JSON parameter output
- 10-20 common sound types

**Expansion (Week 4+):**
- Multiple synths
- Audio preview before saving
- User refinement ("make it brighter")
- Community preset sharing

## Example Interaction

<Tabs>
  <Tab title="User Request">
**User:** "I need a warm analog bass for synthwave, thick and sustained"

**AI thinks:**
- "Warm" → lower filter cutoff, vintage oscillator
- "Analog" → slight oscillator drift, saturation
- "Thick" → multiple oscillators slightly detuned
- "Sustained" → long envelope release
  </Tab>

  <Tab title="Generated Preset">
```json
{
  "presetName": "Warm Synthwave Bass",
  "plugin": "moog-minimoog",
  "parameters": {
    "oscillator1_waveform": "sawtooth",
    "oscillator1_octave": -1,
    "oscillator2_waveform": "sawtooth",
    "oscillator2_octave": -1,
    "oscillator2_detune": 5,
    "filter_cutoff": 35,
    "filter_resonance": 25,
    "filter_type": "lowpass",
    "envelope_attack": 10,
    "envelope_decay": 30,
    "envelope_sustain": 80,
    "envelope_release": 400,
    "saturation": 15
  },
  "description": "Warm analog bass with thick sustain, perfect for synthwave"
}
```
  </Tab>

  <Tab title="User Feedback">
**User:** "Perfect! Can you make it slightly brighter?"

**AI adjusts:**
```json
{
  "parameters": {
    "filter_cutoff": 45,  // increased from 35
    "filter_resonance": 30  // increased from 25
  }
}
```
  </Tab>
</Tabs>

## Challenges

<Warning>
**Plugin SDK Required:** May not have access to all plugin internals

**Sound Verification:** Can't actually HEAR the preset without audio rendering

**Synth Differences:** "Warm" means different things on different synths

**User Expectations:** Generated preset might not match their mental sound
</Warning>

## Plugin SDK Requirements

**Need from UAD:**
- Parameter schema for target synths (JSON)
- Min/max values for each parameter
- Parameter grouping (oscillator, filter, envelope)
- Preset file format specification

**Optional:**
- Preset rendering API (generate audio preview)
- Direct integration with LUNA/Console

## Cost Estimates

**Expected Usage (Month 1):**
- 500 generations

- Audio preview: 25-50 (if implemented)
